# Создание ТЗ

```
Сгенерируй детальное техническое задание (ТЗ) для MVP программного продукта.
ТЗ должно быть оформлено в Markdown и следовать лучшим современным практикам проектирования b2c и b2b SaaS, а также методологиям DDD и Clean Architecture.
Пиши **компактно**, но **достаточно**, чтобы разработчик мог начать реализацию MVP. Не добавляй лишних разделов. Не повторяй уже сказанное. Не углубляйся в теорию, не приводи примеров кода и т.п..

Структура ТЗ находится в файле @Структура\ ТЗ MVP1.md
Концепция продукта находится в файле @Концепция\ MVP1.md
```
